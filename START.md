## 🧠 Key Systems Explained

### Trust Score (0-100)
```
Score ≥ 70  → GREEN   → Trusted (high limits)
Score 40-69 → YELLOW  → Caution (medium limits)
Score < 40  → RED     → Limited (low limits)
```

### AI Analysis (8 Factors)
The AI checks:
1. Your trust score
2. Payment amount (within limits?)
3. How many payments you make
4. Your device (known to system?)
5. Unusual patterns (suspicious?)
6. Fraud indicators
7. Receiver trusted?
8. You know your PIN?

### Promise Tokens
Every payment creates a unique ID that:
- ✅ Prevents double-spending
- ✅ Works offline
- ✅ Expires in 30 days
- ✅ Includes device signature

### Payment Modes
```
NORMAL PAYMENT    → Regular transfers (standard limits)
EMERGENCY MODE    → Higher limits (+25% boost) for crisis
LEND MONEY        → Special lending feature
SPLIT PAYMENT     → Share cost with multiple people
```

## ❓ FAQ

**Q: Is my data saved?**
A: Yes! Uses browser LocalStorage. Data persists between sessions.

**Q: Does it need internet?**
A: No! Works completely offline. Sync simulates bank connection when needed.

**Q: Can I customize it?**
A: Yes! All customization options in SETUP.md

**Q: Can I deploy it?**
A: Yes! Include server.js for backend, add database, setup HTTPS.

**Q: Is it secure?**
A: Demo is for testing. Production needs HTTPS, encryption, compliance checks.

**Q: What if I mess something up?**
A: Just refresh the page or clear localStorage to reset.

**Q: How do I change the theme?**
A: Edit static/css/style.css color variables at the top.

**Q: How do zones work?**
A: Trust score determines zone (Green/Yellow/Red), which sets transaction limits.

**Q: Why does AI block sometimes?**
A: Read the explanation! It shows all 8 factors it analyzed.

**Q: What's a Promise Token?**
A: Unique ID per transaction that prevents double-spending offline.

**Q: How do I add new features?**
A: Edit HTML (page), CSS (styling), JavaScript (logic).
