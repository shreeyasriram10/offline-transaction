## 🎨 Visual Feedback Guide

### Button Color Meanings

| Color | Meaning | Action |
|-------|---------|--------|
| **Gray/Normal** | Ready to use | Click or press key |
| **Yellow ⚡** | Processing | Wait for result |
| **Green ✓** | Success | Authentication passed |
| **Red ✗** | Failed | Try again |

### Status Text
- "Ready" = Not used yet
- "Verifying..." = In progress
- "Verified ✓" = Successfully authenticated
- "Failed ✗" = Authentication unsuccessful
- "Listen..." = Waiting for keyboard input

## ❓ Frequently Asked Questions

### Q: Can I use keyboard shortcuts anywhere?
**A**: Yes! Press `M` anywhere on the dashboard. Keys 1-4 work when monitor is open.

### Q: What if I don't have 5 keys for keyboard auth?
**A**: All 5 can be the same key! Just press any key 5 times.

### Q: Does emergency mode change forever?
**A**: No, it's per-session. Once used, you need to activate it again next time.

### Q: Can I close the monitor with ESC key?
**A**: Not currently - use `M` key or click the ✕ button.

### Q: What happens if keyboard auth times out?
**A**: After 10 seconds of no input, it fails automatically. Try again!

### Q: Is my location really being checked?
**A**: This is a simulation of location verification. In production, it would use GPS data.

## 🔍 Troubleshooting

### Monitor button not appearing?
- Resize browser to wider screen
- Try keyboard shortcut `M` instead
- Check if monitor is already open (slide back in)

### Keyboard shortcuts not working?
- Click on the page first to ensure it has focus
- Make sure you're using English keyboard layout
- Try on dashboard page specifically

### Toast notifications not showing?
- Look at bottom-left corner of screen
- Check if notifications are being blocked by browser
- Refresh the page

### Emergency mode not working?
- Verify payment.html file exists
- Check browser console for errors
- Try login again
